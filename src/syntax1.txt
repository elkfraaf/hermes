Syntax:
an <ID> is any string without whitespace.
An <smt_script> is a valid smtlib script (e.g., it can be given to an SMT solver).
An <smt_term> is a valid smtlib term.

<graph> := (graph <ID> <node>* <edge>* )
<node> := ( node <node_type> <ID>)
<node_type> := entailment | and | done
<edge> := (edge <ID> <src> <dest> <attribute>*)
<src> := __ | ID.<port>
<dest> := __ | ID.<port>
<port> := <entailment_port> | in | out
<entailment_port> := base | kb | g | valid | cmodel  
<attribute> := <smt25_attribute> | <boolX_attribute> | <evaluate_attribute>
<smt25_attribute> := (smt25 (<smt_script>))
<boolX_attribute> := (boolX (<boolX_value>))
<boolX_value> := true | false | unknown | __
<evaluate_attribute> := (evaluate (<get-value>*))
<get-value> := (get-value <term>)


Comments:
-For an edge (<ID> <src> <dest> <attribute>*) such that src (dest) has the form ID.port:
    -if ID is an identifier of an entailment node, then port is an <entailment_port>
    -if ID is an identifier of an and node, then port is either in or out.
    -If ID is an identifier of a done node, then port is in.
-Edges that go from (to) "__" have no source (destination).
-all nodes definitions appear before all edges definitions
